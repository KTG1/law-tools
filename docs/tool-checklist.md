# Tool release checklist

## Legal and product

- [ ] Purpose, audience, and jurisdiction are explicit.
- [ ] Primary legal sources are cited and reviewed.
- [ ] Effective date and last review date are visible.
- [ ] Assumptions and limitations are documented.
- [ ] Informational-use disclaimer is displayed appropriately.
- [ ] A qualified reviewer has approved high-impact legal rules.

## Engineering

- [ ] Core rules are separated from delivery-target code.
- [ ] Boundary cases and known examples have automated tests.
- [ ] Inputs are validated and outputs are escaped.
- [ ] Accessibility and mobile behavior have been checked.
- [ ] Error monitoring does not expose sensitive user data.

## Privacy and security

- [ ] Data collection is minimized.
- [ ] Retention and deletion behavior are documented.
- [ ] Secrets are excluded from source control.
- [ ] Dependencies and deployment configuration have been reviewed.
